简述#

 本文主要记录创建博客和发布这篇文章的简要过程，由于还要很多不了解的，文章中也可能还有错误，望海涵。

创建步骤#

软件安装#

  • node.js
  • git
  • hexo
    • 可以使用 xxx -version 查看是否安装成功，如果安装成功会返回对应的版本号。(xxx与需要确认的软件有关，如git)

遇到的问题#

hexo deploy 时一直报错deployer not found#

 刚开始以为是git没有安装，卸载后重装也不行。后面找到这个博主的链接，发现需要再安装软件才解决了。
https://blog.csdn.net/qq_41688840/article/details/125826614

FATAL YAMLException: unknown escape sequence (21:12)#

 配置文件_config.yml修改错误，参考后面的log可以找到具体错误点。

发布文章发布的是hello world,而不是自己修改后测试博客#

 hexo generate 命令默认会对source文件夹下面的所有.md文件渲染，无法指定。可以添加**–watch**命令让其监视文件变动并立即重新生成静态文件。只有变动的文件才会写入

上传后访问GitHub对应的显示404#

 github对应的库没有设置成public

使用记录#

创建新文章#

  • 使用markdown方式在xx/sources/_posts/下创建一篇.md文件
  • 终端中输入以下命令：
    • hexo s: 启动local的服务器，可以预览生成的文章
    • hexo g: 对所有的md文件编译生成html静态文件，可以在 http://localhost:4000/中预览
      • 在更新了相关文章或者网页配置的时候使用
    • hexo d: 上传到github

修改配置文件后博客主题没有变化#

  • 确保修改的配置文件是正确的，有2个_config.yml文件，一个路径就在当前生成博客，与source public等文件夹在同一级，还有一个则在themes/butterfly下面，2个文件配置的内容不同
    • 当前博客目录下的配置文件：
      • 文章语言，分类，标签等
    • themes下面的配置文件：
      • 封面图等
      • 具体可配置内容参考默认的配置文件
    • 需要注意2个配置文件的默认配置项是不一样的，不能混用
  • 如果确认修改的文件是正确的，可以尝试关闭之前的hexo server，重新打开

最终生成的网页与markdown格式预览不符#

  • 应用主题导致的，主题定义了其他格式
  • #或者+等符号后面没加空格

使用Hexo-Blog-Encrypt插件添加密码后查看还是不需要密码#

  • 关闭之前的hexo server命令，然后使用hexo clean; hexo -s g重新生成文件
  • 之前浏览器已经输入一次密码，后续自己虽然加密都不用再输入密码了。可以通过生成过程中有没有下图log判断是否加密成功
