使用标准延迟格式（SDF）

SDF文件内容

 这一章主要介绍SDF文件的内容。对于文件的每个部分，都讨论了用途，指定了语法并提供了一个示例。具体完整的语法在后面的每一个章节中有描述。SDF文件是一种ASICC文本文件。每个SDF文件包括一个头部信息和一个或多个cell项（entry）。

语法示例：
delay_file :== (DELAYFILE sdf_header cell+)

 头部信息，sdf_header包含和整个文件相关的所有信息，如设计名字，生成SDF工具，和其他参数用来区分设计和操作条件。
 每个cell项，cell标记设计的一部分（”region”或”scope”），并包含用于延时（delay）、时序检查（timing checks）、约束、时序环境（timing environment）的数据。 Cell可以是ASIC库里的物理单元、特定分析工具的模型单元或用户自己创建的部分。实际上，一个cell可以包含整个设计。

sdf_version ::= ( SDFVERSION QSTRING )

eg: (SDFVERSION “OVI 3.0”)

&nbsp;在OVI SDF 2.1 及更高版本中，需SDF版本项和QSTRING字符串。在OVI SDF版本1.0中,SDF版本这一项是必需的，但QSTRING字符串本身可以省略。在OVI SDF版本2.0中，SDF版本这一项和QSTRING字符串都是可选的。SDF OVI 之前的版本不允许SDF版本这一项。
&nbsp;即使SDF版本这一项是可选的，在生成SDF文件时也推荐包含这一项。如果这一项存在，整个文件要严格遵守对应的SDF版本规定的格式
&nbsp;SDF文件的阅读器也可以使用SDF版本这一项来适应不同版本之间文件语法的差异。如果文件不包含SDF版本这项，或者存在，但QSTRING字符串为空，那么阅读器对语法差异的操作是未定义的，如果阅读器无法自动适应所使用的SDF版本的语法，则可能会导致意外错误。
### design name
&nbsp;用于指定SDF文件中记录的时序数据适用的设计名称。
